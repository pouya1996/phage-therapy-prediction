# Phage Therapy Prediction System

A machine-learning pipeline that predicts **phage–host interactions** to guide
bacteriophage therapy.  Given the genome of a clinical bacterial isolate
(FASTA), the system ranks candidate therapeutic phages by their predicted
**Cocktail Interaction (CI) score** — the probability of a productive
interaction at each tested concentration.

## Features

- **7 models** trained and evaluated in a single pipeline (KNN, SVM, Random
  Forest, XGBoost, AdaBoost, Logistic Regression, Multiview CNN)
- **3-step training**: 10-fold cross-validation → held-out test evaluation →
  retrain on all data for production
- **Web UI**: React dashboard with interactive radar charts, bar charts,
  confusion matrices, and per-model prediction tabs
- **REST API**: FastAPI backend for programmatic access
- **Automated feature extraction**: DNA FASTA → ORF finding → protein
  translation → 27-feature vectors (amino-acid composition + physicochemical
  properties + molecular weight) aggregated with 6 statistics

## Quick Start

The fastest way to get running (assuming Python 3.10+ and Node.js 18+ are
installed):

```bash
# 1. Setup
./setup.sh

# 2. Prepare features from raw FASTA files
source .venv/bin/activate
python scripts/prepare_dataset.py

# 3. Train all models
python scripts/train_models.py

# 4. Launch the app (backend + frontend)
./run.sh
```

Open **http://localhost:3000** in your browser.

### 2. Data Preparation

The system ships with pre-built training data generated by `data_cleaner.py`
from the original research paper. **If you just want to use the included
models for prediction, skip to [Step 3](#3-training).**

> **⚠️ Disclaimer — Morphology data:** In this project, the morphology
> labels assigned to phages (Myoviridae, Podoviridae, Siphoviridae) have
> been **generated randomly** for demonstration purposes. For any real-world
> or production use, these labels must be **validated and revised** based on
> the true taxonomic classification of each phage.

If you want to **retrain with your own data**, follow the steps below.

#### 2a. Provide interaction data

You must supply a file called `data/interactions_data.csv` with the following
columns:

| Column | Type | Description |
|--------|------|-------------|
| `phage` | string | Phage genome accession (must match a FASTA filename in `data/raw/phages/`) |
| `host` | string | Host genome accession (must match a FASTA filename in `data/raw/clinical_isolates/`) |
| `class` | int | Interaction label: `1` = lysis (positive), `0` = no lysis (negative) |
| `dataset` | string | Split label: `train` or `test` |
| `morphology` | string | Phage morphology family (e.g. `Myoviridae`, `Podoviridae`, `Siphoviridae`) |
| `concentration` | float | Phage concentration used (e.g. `0.001`) |

Example rows:

```csv
phage,host,class,dataset,morphology,concentration
KT591076,CP027541,1,train,Siphoviridae,0.001
MH051254,NBSQ01000002,0,train,Podoviridae,0.001
NC_028851,CP023483,0,test,Myoviridae,0.001
```

> **Tips for building your own interaction file:**
> - Assign roughly 70 % of unique hosts to `train` and 30 % to `test` to
>   avoid data leakage between splits.
> - Balance positive and negative examples within each split.
> - If you don't know the morphology, use one of the three families listed
>   above based on available phage metadata.

#### 2b. Place raw FASTA files

Put your genome files in the corresponding directories:

```
data/raw/
├── clinical_isolates/    ← bacterial host genomes (.fasta)
└── phages/               ← bacteriophage genomes (.fasta)
```

#### 2c. Extract features

Run the feature extraction pipeline:

```bash
source .venv/bin/activate
python scripts/prepare_dataset.py
```

This script performs two tasks:

**1. Sync phage metadata** — reads every unique phage and its morphology from
`interactions_data.csv` and adds any missing entries to
`data/phage_library/phage_metadata.json`.  You do **not** need to create or
edit `phage_metadata.json` manually.

**2. Extract features** — processes each FASTA file through:
1. **ORF finding** — identifies open reading frames in DNA
2. **Protein translation** — translates ORFs to amino-acid sequences
3. **Feature extraction** — computes 27 features per protein:
   - 21 amino-acid composition fractions
   - 5 elemental composition values (C, H, O, N, S)
   - 1 molecular weight
4. **Aggregation** — summarises per-protein features into 6 × 27 matrices
   (mean, max, min, std, var, median)

Output is saved to `data/processed/features/`.

> **Note:** `prepare_dataset.py` does **not** generate `interactions_data.csv`.
> You must create that file yourself before running this script (see Step 2a).
> However, `phage_metadata.json` is auto-generated from it — no manual
> editing required.

### 3. Training

Train all 7 models with a single command:

```bash
python scripts/train_models.py
```

Or train a specific model:

```bash
python scripts/train_models.py --model rf          # Random Forest only
python scripts/train_models.py --model cnn         # Multiview CNN only
python scripts/train_models.py --model xgboost     # XGBoost only
python scripts/train_models.py --cv-folds 5        # Custom CV folds
```

**Available model names:** `knn`, `svm`, `rf`, `xgboost`, `adaboost`, `lr`,
`cnn`

#### What happens during training

For each model the pipeline executes three steps:

| Step | Purpose | Data used |
|------|---------|-----------|
| 1. K-Fold CV | Evaluate generalisation (per-fold metrics) | Training set only |
| 2. Test evaluation | Final held-out performance | Train on training set, evaluate on test set |
| 3. Production retrain | Save the model used for inference | **All data** (train + test) |

### 5. Making Predictions

#### Via the Web UI

1. Open http://localhost:3000
2. Navigate to the **Predict** tab
3. Upload a clinical-isolate FASTA file (`.fasta`, `.fa`, `.fna`, `.fsa`)
4. Configure settings:
   - **View**: *Best per Phage* (unique phages, best score) or
     *All Interactions* (every phage × concentration combo)
   - **Top K**: Number of top candidates to return
   - **Threshold**: Minimum CI score for a "viable" candidate
5. Click **Run Prediction**
6. Browse per-model results using the model tabs
7. Download rankings (CSV) or reports (TXT) per model

#### Via the API

```bash
curl -X POST http://localhost:8000/api/predict \
  -F "file=@your_isolate.fasta" \
  -F "top_k=10" \
  -F "view=interaction" \
  -F "threshold=0.5"
```

Response structure:

```json
{
  "models": {
    "rf": {
      "rankings": [
        {
          "rank": 1,
          "phage_id": "BP009",
          "morphology": "Siphoviridae",
          "concentration": 100,
          "ci_score": 0.8123,
          "feasible": true
        }
      ],
      "report": "..."
    },
    "multiview_cnn": { "..." },
    "knn": { "..." }
  },
  "metadata": {
    "isolate": "your_isolate.fasta",
    "total_interactions": 70,
    "models_used": ["knn", "svm", "rf", "xgboost", "adaboost", "lr", "multiview_cnn"]
  }
}
```

#### Via Python

```python
from pathlib import Path
from src.prediction.preprocessor import PredictionPreprocessor
from src.prediction.predictor import PhagePredictor
from src.prediction.ranker import PhageRanker

# 1. Extract features from clinical isolate
preprocessor = PredictionPreprocessor()
isolate_features = preprocessor.process_new_isolate(Path("isolate.fasta"))
X_cnn, X_mlp, X_baseline, info = preprocessor.prepare_prediction_input(isolate_features)

# 2. Load models and predict
predictor = PhagePredictor()
predictor.load_all_models()
results = predictor.predict_all_models(X_cnn, X_mlp, X_baseline)

# 3. Rank phages per model
for model_name, probs in results.items():
    ranker = PhageRanker(info, probs, model_name=model_name)
    top5 = ranker.get_top_k_phages(k=5, view="interaction")
    print(f"\n{model_name}:")
    print(top5.to_string())
```

## Models

### Baseline Models

All baseline models use flattened 328-dimensional feature vectors
(6 × 27 × 2 proteomic features + 3 one-hot morphology + 1 log-concentration).
Each model has its own `StandardScaler` fitted during training.

| Model | Key hyperparameters |
|-------|---------------------|
| **KNN** | k=5, distance-weighted |
| **SVM** | RBF kernel, C=1.0, probability calibration |
| **Random Forest** | 100 trees, max_depth=10 |
| **XGBoost** | 100 estimators, lr=0.1 |
| **AdaBoost** | 50 estimators |
| **Logistic Regression** | max_iter=1000, L2 regularisation |

### Multiview CNN

Two-branch neural network:

- **CNN branch**: processes proteomic feature matrices (6, 27, 2) — one channel
  per organism (phage / host). Uses Conv2D → BatchNorm → MaxPool → Flatten.
- **MLP branch**: processes morphology + concentration features (4,).
- **Fusion**: branches are concatenated → Dense layers → sigmoid output.

Trained with Adam optimiser, early stopping (patience=20), and a separate
`StandardScaler` for the CNN input.

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health` | Server status and loaded model list |
| `GET` | `/api/models` | Available model names |
| `GET` | `/api/phages` | All phages in the library (with morphology) |

| `GET` | `/api/results` | Training evaluation metrics (CV + test, per model) |
| `GET` | `/api/training-info` | Dataset statistics (sample counts, distributions) |

### POST /api/predict

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `file` | File | *(required)* | Clinical-isolate FASTA file |
| `top_k` | int | 10 | Number of top candidates per model |
| `view` | string | `"phage"` | `"phage"` (all combos) or `"interaction"` (best per phage) |
| `threshold` | float | 0.5 | CI-score feasibility threshold |

## Configuration

```yaml
# Model hyperparameters
models:
  rf:
    n_estimators: 100
    max_depth: 10
  cnn:
    epochs: 100
    batch_size: 32
    patience: 20

# Training
training:
  cv_folds: 10
  random_seed: 42

# File paths
paths:
  data_raw: 'data/raw/'

  port: 8000

logging:
  level: 'INFO'              # DEBUG for verbose output
```

## Troubleshooting

### Common issues

| Problem | Solution |
|---------|----------|
| `ModuleNotFoundError` | Activate the virtual environment: `source .venv/bin/activate` |
| `No trained models found` | Run `python scripts/train_models.py` before launching the app |
| `Feature file not found` | Run `python scripts/prepare_dataset.py` to generate features |
| Port already in use | Run `./run.sh --stop` then start again |
| TensorFlow warnings | These are informational and can be ignored |
| Slow first prediction | Model loading happens on first request; subsequent predictions are fast |

### Checking logs

```bash
# Backend log (when started via run.sh)
tail -f .backend.log

# Frontend log
tail -f .frontend.log

# Set verbose logging
# Edit config.yaml → logging.level: 'DEBUG'
```
